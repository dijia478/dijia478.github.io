SpringBoot的使用

发表于 2018-10-06 | 分类于 Spring Boot

来新公司半年多了，在这里第一次接触到Spring Boot，它是一个快速构建spring项目的框架。期间从各种网上资料，同事的指教，还有书籍中，积累了一些使用的心得，后面陆陆续续会整理一下，温故而知新。（我每次都会使用最新的Spring Boot稳定版作为演示，目前是2.0.5）

在刚开始学习Spring这个框架的时候，印象最深刻的就是其繁杂的配置。而Spring Boot最大的特点呢，就是自动配置。因为Spring Boot中提供了很多的条件注解，可以根据这些条件注解完成自动配置！Spring Boot中提倡“零”配置，就是不需要*.xml文件。同时它也集成很多的目前相对比较主流的第三方技术，我们只需要导入对应的Spring Boot和第三方技术整合的starter的maven依赖，Spring Boot就可以完成自动配置，我们只需要在我们的项目中直接使用就可以了!

快速入门

废话不多说，直接上实践，在这里，我来搭建一个基础的Spring Boot项目，并实现简单的Http请求处理，通过这个例子对Spring Boot有一个初步的了解，并体验其结构简单、开发快速的特性。

#三个问题

Spring Boot项目如何做到不需要xml配置文件的？

Spring 3 以后提供了两个注解来替换xml文件的配置:

@Configuration(用来替换applicationContext.xml)

@Bean就相当于我们在applicationContext.xml文件中的\标签

Spring Boot项目如何做到自动配置？

这里以上面的web配置为例。还记得我们在构建项目的第4步，添加了一个spring-boot-starter-web这个maven依赖。那么根据maven的依赖传递特性，会把这个依赖中的其他依赖一并导入到我们的工程中，比如springmvc的依赖包以及tomcat的依赖包。

当我们导入了spring-boot-starter-web这个依赖了以后，还有一个比较重要的依赖包：spring-boot-autoconfigure这个依赖，这个依赖包就是Spring Boot针对第三方的一些主流技术提供的自动配置的包。而web项目需要配置的DispatcherServlet，就是在其中的web包下的DispatcherServletAutoConfiguration类中。通过源码我们看到在DispatcherServletAutoConfiguration中存在一些ConditionalXXX的一些注解，这些注解就是Spring Boot提供的条件注解！当满足某种条件的时候，OK启动配置

Spring Boot项目启动的时候到底加载了哪些自动配置的类?

加载的是spring-boot-autoconfigure包中的META-INF/spring.factories这个文件中的自动配置类。

当然，也是可以自定义自动配置的，只需要在工程中src/main/resources目录下创建一个META-INF/spring.factories的文件，在这个文件中注册我们的自动配置就可以了，例如：

org.springframework.boot.autoconfigure.EnableAutoConfiguration=\
cn.dijia478.hello.config.HelloAutoConfig

四个小知识点

  • 更改默认配置：Spring Boot项目中存在很多的默认配置，比如：tomcat的端口号的配置，默认使用的是8080，如果我们想更改这个默认配置，可以在Spring Boot给我们提供的一个全局配置文件中进行更改，这个全局配置文件的路径是在：src/main/resources下，这个文件的名称是: application.properties，比如修改tomcat的端口号为8888: server.port=8888。（这里本人比较推荐使用application.yml的格式，结构会更清晰）
  • spring-boot-starter-XXX：Spring Boot官方提供了很多的这种格式的pom，这些pom中定义了指定的技术的一些依赖，比如，spring-boot-starter-web
  • 关闭指定的自动配置：@SpringBootApplication(exclude = {DispatcherServletAutoConfiguration.class})
  • @ImportSource注解：Spring Boot项目提倡的是“零”配置，但是在某些场景下我们可能还需要使用xml文件来完成我们的配置，那么就可以使用该注解来加载我们的配置文件
